# Execution plans

Use an execution plan for work that spans multiple subsystems, changes public
contracts, or needs several rounds of evidence. Small local fixes can stay in
the task conversation.

An active plan contains:

- `Status: active` and one concrete objective;
- current authoritative evidence, not intended outcomes;
- decisions and rejected alternatives;
- ordered remaining work and objective exit criteria;
- dated progress entries when the direction changes.

Store active plans in `active/`. Move a plan to `completed/` only when every exit
criterion is proven by current repository or external evidence. Do not mark an
external publishing/adoption requirement complete from local configuration.
